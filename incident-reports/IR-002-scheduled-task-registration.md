# IR-002: Scheduled Task Registration Detected

## Incident Summary

A Wazuh level 9 alert detected the registration of a scheduled task on the monitored Windows endpoint.

The activity was generated as a safe, authorized lab simulation to validate the scheduled-task registration detection rule.

## Incident Details

| Field            | Value                                            |
| ---------------- | ------------------------------------------------ |
| Incident ID      | IR-002                                           |
| Title            | Scheduled Task Registration Detected             |
| Severity         | Medium                                           |
| Status           | Closed - Authorized Test Activity                |
| Detection Date   | 21 June 2026                                     |
| Endpoint         | `SOC-WIN11-01`                                   |
| Detection Source | Wazuh with Task Scheduler Operational logs       |
| Windows Event ID | `106`                                            |
| Wazuh Rule ID    | `100102`                                         |
| MITRE ATT&CK     | `T1053.005` - Scheduled Task/Job: Scheduled Task |

## Detection Logic

The alert was generated when Wazuh detected a scheduled task registration event from the Task Scheduler Operational event channel.

The custom Wazuh rule builds on the built-in task-registration rule and raises a higher-priority alert for analyst investigation.

## Triggering Command

The following safe command was intentionally executed in the authorized lab:

```powershell
schtasks /Create /TN "SOC-Lab-Persistence-Test-2" /TR "cmd.exe /c echo PersistenceTest2" /SC ONCE /ST 23:57 /F
```

## Alert Evidence

The Wazuh alert contained:

- Endpoint hostname
- User context
- Scheduled task name
- Task Scheduler Operational event channel
- Windows Event ID `106`
- Wazuh Rule ID `100102`
- Alert severity level `9`
- MITRE ATT&CK mapping to `T1053.005`

## Investigation Process

1. Reviewed the Wazuh alert generated by Rule `100102`.
2. Confirmed the event originated from `Microsoft-Windows-TaskScheduler/Operational`.
3. Verified Windows Event ID `106`, indicating scheduled task registration.
4. Reviewed the task name, user context, endpoint identity, and event details.
5. Confirmed the task was created manually as an authorized detection-validation test.
6. Confirmed the task action was benign and did not execute malware, download content, modify persistence locations, or perform external network activity.

## Root Cause

The alert was triggered by an intentional lab simulation that created the scheduled task `SOC-Lab-Persistence-Test-2`.

## Impact Assessment

No security impact was identified.

The scheduled task was created only to validate detection coverage. Its configured action was a harmless `cmd.exe` command that echoed test text. No persistence mechanism was retained for operational use, and no malicious payload or unauthorized system modification occurred.

## Response Actions

- Confirmed the event was authorized.
- Documented the alert as a successful scheduled-task detection validation.
- Retained Rule `100102` because scheduled task registration is a useful persistence and execution investigation signal.
- Planned removal of the lab-created scheduled tasks as cleanup.
- Closed the incident as authorized test activity.

## Analyst Notes

Scheduled task registration is not inherently malicious. In a real enterprise environment, an analyst should investigate:

- Task name, folder, and description
- Executable, script, or command configured to run
- Trigger type, timing, and recurrence
- Creator account and endpoint context
- Whether the task runs with elevated privileges
- Whether PowerShell, temporary files, user-profile paths, or unusual binaries are involved
- Whether similar tasks were created on other endpoints
- Whether the activity is approved or associated with enterprise management tooling

## Cleanup Verification

The lab-created scheduled tasks were removed after detection validation:

- `SOC-Lab-Task-Test`
- `SOC-Lab-Persistence-Test`
- `SOC-Lab-Persistence-Test-2`

Cleanup was verified using `schtasks /Query`, confirming that the test tasks no longer existed on the endpoint.

## Lessons Learned

- Task Scheduler Operational Event ID `106` provides useful scheduled-task registration telemetry.
- Wazuh can collect and alert on Task Scheduler Operational events from Windows endpoints.
- Custom detection rules can build on built-in Wazuh rules to improve alert severity and investigation visibility.
- Scheduled task creation is a persistence-relevant signal, but analyst validation is required before classifying it as malicious.
- Authorized simulations should be documented and cleaned up after validation.
