# IR-004: Local Account Creation Detected

## Incident Summary

A Wazuh level 10 alert detected the creation of a local user account on the monitored Windows endpoint.

The activity was generated as a safe, authorized lab simulation to validate the local account-creation detection rule.

## Incident Details

| Field            | Value                             |
| ---------------- | --------------------------------- |
| Incident ID      | IR-004                            |
| Title            | Local Account Creation Detected   |
| Severity         | High                              |
| Status           | Closed - Authorized Test Activity |
| Detection Date   | 21 June 2026                      |
| Endpoint         | `SOC-WIN11-01`                    |
| Detection Source | Windows Security Event Log        |
| Windows Event ID | `4720`                            |
| Wazuh Rule ID    | `100104`                          |
| MITRE ATT&CK     | `T1098` - Account Manipulation    |

## Triggering Activity

The following authorized lab command created the test account:

```powershell
net user SOCLabTemp3 "TempLabPass!2026" /add
```

## Alert Evidence

The Wazuh alert showed:

- Endpoint hostname
- Created account name: `SOCLabTemp3`
- Creating account: `socanalyst`
- Windows Event ID `4720`
- Wazuh Rule ID `100104`
- Alert severity level `10`
- MITRE ATT&CK mapping to `T1098`

## Investigation Process

1. Reviewed the Wazuh alert generated by Rule `100104`.
2. Confirmed Windows Security Event ID `4720`.
3. Verified the created account name and creating user context.
4. Confirmed the account was intentionally created as an authorized lab test.
5. Confirmed no privileged group membership, remote access use, or malicious follow-on activity was associated with the account.
6. Planned removal of all temporary lab accounts after validation.

## Root Cause

The alert was triggered by an intentional lab simulation that created temporary local accounts for detection validation.

## Impact Assessment

No security impact was identified.

The created accounts were temporary lab-only accounts. They were not used for unauthorized access, privilege escalation, persistence, or remote execution.

## Response Actions

- Confirmed the event was authorized.
- Documented the alert as a successful detection validation.
- Retained Rule `100104` because local account creation is a high-value identity and persistence signal.
- Planned removal of all temporary lab accounts.
- Closed the incident as authorized test activity.

## Cleanup Verification

The following test accounts were removed after validation:

- `SOC-Lab-TempUser`
- `SOCLabTemp2`
- `SOCLabTemp3`

Cleanup was verified by querying the local user database after deletion.

## Lessons Learned

- Windows Security Event ID `4720` provides reliable local account-creation telemetry.
- Built-in Wazuh rules can be refined with custom event-ID filtering.
- User identity, target account name, event ID, and follow-on activity are important triage details.
- High-severity account-creation alerts require validation before escalation.
- Authorized identity simulations should be removed after testing.
