# IR-003: Windows Startup Folder File Creation Detected

## Incident Summary

A Wazuh level 9 alert detected a file being added to the Windows Startup folder on the monitored endpoint.

The activity was generated as a safe, authorized lab simulation to validate File Integrity Monitoring and the custom Startup-folder detection rule.

## Incident Details

| Field            | Value                                            |
| ---------------- | ------------------------------------------------ |
| Incident ID      | IR-003                                           |
| Title            | Windows Startup Folder File Creation Detected    |
| Severity         | Medium                                           |
| Status           | Closed - Authorized Test Activity                |
| Detection Date   | 21 June 2026                                     |
| Endpoint         | `SOC-WIN11-01`                                   |
| Detection Source | Wazuh File Integrity Monitoring                  |
| Wazuh Rule ID    | `100103`                                         |
| Parent Rule      | `554`                                            |
| MITRE ATT&CK     | `T1547.001` - Registry Run Keys / Startup Folder |

## Detection Logic

The alert was generated when Wazuh detected a newly added file within:

```text
C:\ProgramData\Microsoft\Windows\Start Menu\Programs\Startup
```

The custom rule elevated the event because files in this directory may execute automatically at user logon.

## Triggering Activity

The following authorized lab command created the test file:

```powershell
Set-Content -Path "C:\ProgramData\Microsoft\Windows\Start Menu\Programs\Startup\SOC-Lab-Startup-Test-3.txt" -Value "Authorized startup-folder FIM validation"
```

## Alert Evidence

The Wazuh alert showed:

- Endpoint hostname
- Startup-folder file path
- Wazuh Rule ID `100103`
- Alert severity level `9`
- MITRE ATT&CK tactic and technique mapping
- Persistence-relevant file location

## Investigation Process

1. Reviewed the Wazuh event generated by Rule `100103`.
2. Confirmed the file path was inside the Windows Startup folder.
3. Reviewed the alert severity, path, endpoint context, and MITRE mapping.
4. Confirmed the file was created manually as part of an authorized FIM validation test.
5. Verified that the file was a harmless text file and did not execute code, download content, or create additional persistence.

## Root Cause

The alert was triggered by an intentional lab simulation that created a harmless text file in the Windows Startup folder.

## Impact Assessment

No security impact was identified.

The test file was created only to validate monitoring and detection coverage. It did not contain executable code and did not create an active persistence mechanism.

## Response Actions

- Confirmed the activity was authorized.
- Documented the event as a successful FIM detection validation.
- Retained Rule `100103` because Startup-folder changes are high-value persistence signals.
- Planned removal of all test files created during FIM validation.
- Closed the incident as authorized test activity.

## Cleanup Verification

The following lab files were removed after validation:

- `SOC-Lab-Startup-Test.txt`
- `SOC-Lab-Startup-Test-2.txt`
- `SOC-Lab-Startup-Test-3.txt`

Cleanup was verified by confirming that no matching files remained in the Startup folder.

## Lessons Learned

- Wazuh FIM can monitor Windows paths in real time.
- Built-in file-added alerts can be extended with custom rules for persistence-relevant locations.
- The Windows Startup folder is a useful persistence monitoring target.
- File creation in a sensitive path requires analyst validation before it is classified as malicious.
- Lab simulations should be cleaned up after detection validation.
