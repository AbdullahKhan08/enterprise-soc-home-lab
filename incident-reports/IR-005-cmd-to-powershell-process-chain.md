# IR-005: PowerShell Launched by cmd.exe Detected

## Incident Summary

A Wazuh level 8 alert detected PowerShell being launched by `cmd.exe` on the monitored Windows endpoint.

The activity was generated as a safe, authorized lab simulation to validate suspicious parent-child process-chain detection.

## Incident Details

| Field            | Value                                   |
| ---------------- | --------------------------------------- |
| Incident ID      | IR-005                                  |
| Title            | PowerShell Launched by cmd.exe Detected |
| Severity         | Medium                                  |
| Status           | Closed - Authorized Test Activity       |
| Detection Date   | 21 June 2026                            |
| Endpoint         | `SOC-WIN11-01`                          |
| Detection Source | Sysmon Event ID 1 collected by Wazuh    |
| Wazuh Rule ID    | `100105`                                |
| Parent Rule      | `100100`                                |
| MITRE ATT&CK     | `T1059.001` - PowerShell                |

## Triggering Activity

The following authorized lab command was executed:

```powershell
cmd.exe /c powershell.exe -NoProfile -Command "Get-Date"
```

## Alert Evidence

The Wazuh alert showed:

- Endpoint hostname
- PowerShell command line
- Parent image: `C:\Windows\System32\cmd.exe`
- Parent command line
- PowerShell executable path
- Wazuh Rule ID `100105`
- Severity level `8`
- MITRE ATT&CK mapping to `T1059.001`

## Investigation Process

1. Reviewed the alert generated by Rule `100105`.
2. Confirmed Sysmon Event ID `1` process-creation telemetry.
3. Verified that the parent process was `cmd.exe`.
4. Reviewed the PowerShell command line and parent command line.
5. Confirmed the activity was an authorized lab simulation.
6. Verified that the command executed only `Get-Date` and did not download content, create persistence, modify accounts, or initiate suspicious network activity.

## Root Cause

The alert was triggered by an intentional lab simulation designed to validate parent-child process-chain detection.

## Impact Assessment

No security impact was identified.

The activity used a benign PowerShell command and did not include malware, persistence, credential access, lateral movement, or external network activity.

## Response Actions

- Confirmed the event was authorized.
- Documented the alert as a successful detection validation.
- Retained Rule `100105` because parent-child process relationships provide useful context during PowerShell investigations.
- Closed the incident as authorized test activity.

## Lessons Learned

- Sysmon Event ID `1` provides valuable process ancestry and command-line telemetry.
- Parent-child process analysis improves context beyond a simple PowerShell execution alert.
- `cmd.exe → powershell.exe` should be triaged in context rather than automatically treated as malicious.
- Detection chaining can build higher-context alerts from earlier detections.
- Process-chain evidence is useful for SOC analysts when reconstructing execution flow.
